# Security Policy

LUNORA is built with strict cryptographic controls, double-submit cookie CSRF protections, and Zod configuration validations. We take security seriously.

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.0.x | :white_check_mark: |
| < 1.0.0 | :x: |

## Reporting a Vulnerability

If you discover a security vulnerability in LUNORA, please do **NOT** open a public GitHub issue. Instead, report it through the following process:

1. Send an email to `security@lunora.com` with a detailed description of the vulnerability.
2. Include steps to reproduce the issue, potential impact, and details about the test environment.
3. We will acknowledge receipt of your report within 48 hours and work with you to coordinate a security patch before public release.
